WEBPACK DE ANGULAR:Empaquetador de modulos BUNDLE.JS para compilar un proyecto


La instalación del angular cli debe ser global para ser instalado desde cualquier directorio


CARPETA APP: crear componentes, directivas,etc


ANGULAR JSON: configuracion de archivos js


PACKAJE.JSON: gestiona las versionas de los compiladores


ts.config: gestion de Emacscripts


APP MODULE: declaramos la instancia para que este disponible en toda la aplicacion:

- declaration: van los componentes

- Import Module: van los modulos o logica 


APP-ROOT: Es un componente padre de la aplicacion


Templates: 

- parent component(componente padre): Se implementa Property binding

- parent template(child component): Se implementa Event binding


Data Bindings: Comunicacion entre el .ts y .html

- interpolation: doble comillas(data)

- property binding = [propiedad]="data"

 

Directivas:ng g c,etc


DECORADOR: @component ,@input


component.ts: es una clase que viene declarado con una anotacion @component

declarada al inicio como variable y esta viene ser de tipo de la libreria angular core


ROUTER OULET: importa el array de componentes para estar en la url


EL PRIMER ELEMENTO EN EJCUTARSE EN UN COMPONENT ES EL CONSTRUCTOR, LUEGO EL NGONINIT


INPUT: es un evento para pasarle data del padre al hijo


INJECCION DE DEPENDENCIAS: atraves del constructor invocar datos de otras clases
